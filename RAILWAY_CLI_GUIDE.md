# Railway CLI Quick Start - PSM Enterprise

## Installation

```bash
# macOS
brew install railway

# Linux / WSL
curl -fsSL https://railway.app/install.sh | bash

# Verify installation
railway --version
```

---

## Quick Deployment

### 1. Login to Railway
```bash
railway login
# Opens browser to authenticate with GitHub
```

### 2. Create Project
```bash
# Navigate to your project directory
cd psm-enterprise-fixed

# Initialize Railway project
railway init
# Follow prompts to create new project
```

### 3. Add Services

#### PostgreSQL Database
```bash
railway add
# Select "Database" → "PostgreSQL"
# Railway creates automatically
```

#### Redis Cache
```bash
railway add
# Select "Database" → "Redis"
```

### 4. Set Environment Variables

#### Create `.env.railway` file
```bash
cat > .env.railway << 'EOF'
# Database (from Railway PostgreSQL service)
PG_HOST=${{ Postgres.RAILWAY_PRIVATE_DOMAIN }}
PG_PORT=5432
PG_DATABASE=${{ Postgres.PGDATABASE }}
PG_USER=${{ Postgres.POSTGRES_USER }}
PG_PASSWORD=${{ Postgres.POSTGRES_PASSWORD }}

# Redis (from Railway Redis service)
REDIS_CONNECTION=redis://default:${{ Redis.REDIS_PASSWORD }}@${{ Redis.RAILWAY_PRIVATE_DOMAIN }}:6379

# Application Configuration
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://+:3000
JWT_SECRET=$(openssl rand -hex 32)
JWT_EXPIRES_IN=24h
CORS_ORIGINS=https://your-frontend-domain.railway.app

# Admin Credentials (CHANGE IMMEDIATELY AFTER LOGIN)
ADMIN_SEED_USERNAME=admin
ADMIN_SEED_PASSWORD=$(openssl rand -base64 12)

# License
PSM_LICENSE_SECRET=$(openssl rand -hex 16)

# AI Integration (optional)
ANTHROPIC_API_KEY=
AI_MODEL=claude-sonnet-4-6
AI_ENABLED=true
EOF
```

### 5. Deploy Backend Service

```bash
# Switch to backend context
railway service backend

# Set the Dockerfile
railway env set RAILWAY_DOCKERFILE_PATH=Dockerfile.backend-csharp

# Deploy
railway up

# View logs
railway logs --tail 100
```

### 6. Deploy Frontend Service

```bash
# Create new service for frontend
railway service frontend

# Set the Dockerfile
railway env set RAILWAY_DOCKERFILE_PATH=Dockerfile.frontend

# Set build arguments
railway build-args set VITE_API_URL=/api

# Deploy
railway up

# View logs
railway logs --tail 100
```

## Common Railway CLI Commands

```bash
# View all services
railway service list

# Switch between services
railway service <service-name>

# View environment variables
railway env

# Set a variable
railway env set KEY=value

# Unset a variable
railway env unset KEY

# View deployment logs
railway logs --tail 50

# View real-time logs
railway logs -f

# Open Railway dashboard
railway open

# Open logs in browser
railway logs --browser

# View service status
railway status

# Redeploy service
railway redeploy

# Stop service
railway stop

# Pause service
railway pause

# Resume service
railway resume

# Link services (networking)
railway link <service-name>

# View service metrics
railway metrics

# Debug: SSH into service
railway shell

# Export configuration
railway config export

# Create backup
railway db backup
```

## Environment Variable Reference

### Database Variables
```bash
PG_HOST=<postgres-hostname>
PG_PORT=5432
PG_DATABASE=railway
PG_USER=postgres
PG_PASSWORD=<auto-generated>

# Alternative connection string format
DATABASE_URL=postgresql://postgres:<password>@<host>:5432/railway
```

### Redis Variables
```bash
REDIS_CONNECTION=redis://default:<password>@<hostname>:6379
REDIS_PASSWORD=<auto-generated>
```

### Application Variables
```bash
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://+:3000
JWT_SECRET=<64-char-hex-string>
JWT_EXPIRES_IN=24h
CORS_ORIGINS=https://your-app.railway.app
```

### Admin Setup
```bash
ADMIN_SEED_USERNAME=admin
ADMIN_SEED_PASSWORD=<change-immediately>
PSM_LICENSE_SECRET=<random-string>
```

### Optional Variables
```bash
AI_ENABLED=true
AI_MODEL=claude-sonnet-4-6
ANTHROPIC_API_KEY=<your-api-key>

# Logging
LOG_LEVEL=Information
```

## Troubleshooting with CLI

### View Build Logs
```bash
railway logs --service backend --tail 200
```

### Check Service Health
```bash
railway status
```

### Test Database Connection
```bash
railway shell
# Inside container:
psql -h $PG_HOST -U $PG_USER -d $PG_DATABASE
```

### View Environment in Service
```bash
railway shell
env | grep PG_
env | grep REDIS_
env | grep JWT_
```

### Redeploy After Changes
```bash
# Push code to GitHub first
git push origin main

# Trigger redeploy
railway redeploy

# Watch logs
railway logs -f
```

## Secret Management

### Store Secrets Safely
```bash
# Use Railway's secret management
railway env set JWT_SECRET --secret

# Railway won't display in logs
railway env
```

### Never Commit Secrets
```bash
# .env files should be in .gitignore
echo ".env.local" >> .gitignore
echo ".env.railway" >> .gitignore
```

## CI/CD Integration with GitHub

### Automatic Deployment on Push
1. Railway → Project Settings → **Automatic Deployments**
2. Select GitHub branch (e.g., `main`)
3. Enable auto-deploy on push
4. Railway automatically builds and deploys

## Useful Scripts

### Deploy All Services
```bash
#!/bin/bash

services=("backend" "frontend")

for service in "${services[@]}"; do
  echo "Deploying $service..."
  railway service $service
  railway redeploy
  echo "$service deployed!"
  sleep 5
done

echo "All services deployed!"
```

### Check All Service Status
```bash
#!/bin/bash

services=$(railway service list --json | jq -r '.[].name')

for service in $services; do
  echo "Service: $service"
  railway service $service
  railway status
  echo "---"
done
```

### Export All Environment Variables
```bash
#!/bin/bash

services=$(railway service list --json | jq -r '.[].name')

for service in $services; do
  echo "=== $service ==="
  railway service $service
  railway env
  echo ""
done
```
